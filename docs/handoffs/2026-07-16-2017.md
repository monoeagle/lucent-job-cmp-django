## Entscheidungen dieser Session
- **Kein `rsync`** für die Spiegelung (`rm -rf` + `cp -a` stattdessen): rsync ist
  auf einer minimalen AlmaLinux nicht garantiert vorhanden — für einen
  air-gapped Installer keine akzeptable Abhängigkeit. Preis: es wird jedes Mal
  alles neu kopiert statt inkrementell.
- **`--with-packages` als Opt-in, nicht als Default**: das Bundle muss air-gapped
  funktionieren; diese Zusage darf nicht von freigeschalteten Repos abhängen.
- **Beide PostgreSQL-Varianten erkennen statt eine raten** (Nutzer-Entscheidung).
- **Zertifikat: Rat von ChatGPT bewusst verworfen.** Der Vorschlag war, die reine
  `[ ! -f crt ]`-Prüfung zu behalten — genau der Bug: bei geändertem FQDN liefert
  nginx weiter den alten CN. Jetzt SAN-Abgleich; CA-signierte Zertifikate werden
  nie überschrieben.
- **`cmp`-Guard für systemd-Units abgelehnt**: Restart ist ohnehin nötig (App-Code
  ändert sich), und der Guard verleitet zum Fehlschluss „Unit unverändert → kein
  Restart" — das war der stille Upgrade-Bug.
- **Installer-Arbeit als EIN Commit** statt der vier angebotenen: der Arbeitsbaum
  enthielt nur den Endzustand; vier Commits hätten erfundene Zwischenstände
  bedeutet.
- **Kein Version-Bump** — *im Nachtrag revidiert:* Sobald ein Release gewünscht
  war, **musste** gebumpt werden (v1.1.0 durfte nicht mit anderem Bundle-Inhalt
  überschrieben werden). Wurde **v1.2.0** (MINOR, Policy: autonom).
- **Version-Ableitung statt vierter Stelle:** `run.sh` liest die Version aus
  `lucent-hub.yml`, statt sie zu spiegeln. Eine Stelle, die man nachziehen muss,
  ist eine Stelle, die driftet — genau das war hier passiert.
