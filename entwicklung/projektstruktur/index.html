App-interner Aufbau¶

Jede App unter cmp/apps/ folgt demselben Pattern:

apps/{name}/
├── __init__.py
├── apps.py           # AppConfig
├── models.py         # Django Models
├── services.py       # Business-Logik
├── views.py          # Class-Based Views
├── forms.py          # Django Forms
├── admin.py          # Admin-Registrierung
├── urls.py           # URL-Patterns
└── migrations/       # Auto-generierte Migrationen

Sonderfälle: - provisioning/: zusätzlich tasks.py (Celery) und clients.py (GitLab) - cmdb/: zusätzlich clients.py (CMDB Stub) - accounts/: zusätzlich management/commands/seed.py
