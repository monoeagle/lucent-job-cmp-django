3 · Anhang A — „Neuen Service / neue Bestelloption anlegen“

Der Teil, den du ausdrücklich willst: Kochbuch statt Konzept. Jede Seite ist eine Aufgabe mit Ausgangslage, nummerierten Schritten, kopierbarem Code und einer Probe am Ende („so siehst du, dass es geklappt hat“). Alles hängt an echten Stellen im Code:

A.1 Wie ein Service technisch entsteht

Das Modell in 1 Seite: ServiceTemplate (Name, Kategorie, parameters als JSON) → Formular wird zur Laufzeit aus dem JSON gebaut → Validierung → Bestellung. Wer das verstanden hat, braucht A.2–A.5 nur noch als Rezept.

A.2 Das Parameter-Schema

Feldreferenz mit je einem Beispiel: key, label, type (string/integer/boolean/enum), required, default, group, display_order, constraints.options, depends_on, affects_options_of, tofu_variable_name. Beleg: apps/catalog/services.py:16 (SHARED_PARAMS).

A.3 Rezept: neue Bestelloption zu bestehendem Service

Der häufigste Fall (z. B. „Backup-Zeitfenster“ zur Linux-VM). Schritte: Parameter-Block ergänzen → Gruppe/Reihenfolge setzen → Migration/Seed → im Formular prüfen → Test ergänzen. Mit TDD-Reihenfolge, wie im Projekt Pflicht.

A.4 Rezept: kompletter neuer Service

Z. B. „PostgreSQL-Datenbank“: Template anlegen (Kategorie, Beschreibung, Parameter) → Katalog-Kachel & Detailseite → Wizard-Schritte → Genehmigungsregel → Provisioning-Anbindung (heute Stub) → Seed → Tests. Ende-zu-Ende an einem durchgehenden Beispiel.

A.5 Abhängige Optionen & Validierung

Wenn Auswahl A die Optionen von B einschränkt (depends_on/affects_options_of) — und wie die Regeln in core/domain/validators.py greifen. Inkl. „was passiert bei ungültiger Eingabe“.

A.6 Häufige Fehler

Feld erscheint nicht · Auswahl bleibt leer · Default greift nicht · Parameter kommt nicht in der Bestellung an · vergessener Seed. Je Symptom: Ursache + Prüfbefehl.

4 · Was vorher zu entscheiden / zu prüfen ist

Ungeprüft — kein Rateschluss von mir: Wie importiert euer Bookstack? Es gibt (a) Seite-für-Seite Markdown einfügen, (b) Import über die Bookstack-API mit einem Skript, (c) die eingebaute Import-Funktion neuerer Bookstack-Versionen. Welche Wege eure Instanz kann, hängt von Version und Rechten ab — das muss an eurer Instanz nachgesehen werden, bevor ich das Ausgabeformat festlege. Bis dahin plane ich das robusteste Format: eine Markdown-Datei je Seite, nummeriert in Kapitel-Ordnern — das lässt sich in jedem der drei Wege verwenden.
Offene Punkte, die deine Entscheidung brauchen:
